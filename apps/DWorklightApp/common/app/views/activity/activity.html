Worklight Dynamic Busy Indicator Tests



This tests out the dynamic busy indicator. In hybrid applications, the native Worklight busy indicator is used. For browser based applications, the Dojo Mobile busy indicator emulation is used. IOS devices allow for a custom message to be supplied. For all others, the custom message is ignored. 
require(["dworklight/activity"], function(activity) {
    activity.start("Optional message");
    // alternate calling is: topic.publish("activity/start", "message");
    
    //-- Do something
    
    activity.stop();
    // alternate calling is: topic.publish("activity/stop");
});
The following setting can be applied directly, or passing in an object to activity.init(). 
{
	// defaultMessage : String
	//		Message to show on IOS devices
	defaultMessage : "Loading",

	// useNative : boolean
	//		Use native activity indicator if available.
	useNative   : true,

	// iosOptions : Map
	//		Various options specific to IOS
	iosOptions  : {
		"bounceAnimation" : false,
		"textColor"       : "#FFFFFF",
		"fullScreen"      : false,
		"duration"        : 0,
		"minDuration"     : 2
	},

	// containerId : String
	//		Force specific container for all activities.
	containerId : "",
}
